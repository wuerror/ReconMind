## 主动端口扫描（服务发现）

你正在执行主动扫描阶段。使用 gogo 对前序阶段发现的 IP 进行端口扫描和服务识别。

⚠️ 这是主动操作，会向目标发送网络包。请确保在授权范围内。

### 操作步骤

1. 先 read_file("output/state.json") 获取所有已知 IP
2. 对高价值 IP（已发现有 Web 服务的）优先扫描全端口
3. 对其他 IP 使用 top2 端口集扫描
4. gogo 会自动识别服务指纹，结果自动更新 state.json

### 扫描策略

- **单 IP 深度扫描**：`gogo_scan(target="x.x.x.x", ports="-")` - 全端口，适合高价值目标
- **网段发现扫描**：`gogo_scan(target="x.x.x.0/24", ports="top2")` - 常见端口，适合发现同网段其他资产
- **重点端口精扫**：`gogo_scan(target="x.x.x.x", ports="8000-9999")` - 非标准 Web 端口

### 红队思维

- 前序被动阶段可能遗漏了非标准端口上的服务，主动扫描的价值就在这里
- 同一 IP 的多端口服务往往说明这是一台业务服务器而非 CDN/WAF
- 发现的新服务（如 Redis 6379、MongoDB 27017、Elasticsearch 9200）直接标记为高价值
- 如果发现与被动阶段指纹识别不一致的服务，以主动扫描结果为准

### 完成标志

所有 IP 扫描完毕（gogo 已自动更新 state.json）。输出扫描统计摘要后结束。
